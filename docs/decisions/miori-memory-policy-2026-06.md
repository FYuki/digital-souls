# 光織 Memory Policy

## 目的

光織の長期記憶を、安全かつ実用的に扱うための方針を定義する。

記憶は、人格の一貫性、農業日誌、レシピ管理、日常支援に利用する。

実装が参照する認識語彙・pattern・閾値・検索件数などの機械可読な設定値は
`backend/app/memory/memory_policy.json`で管理する。この文書は人間向けの方針説明であり、
実行時設定のSource of Truthではない。RAG privacyの不変条件と、設定でも緩和できない
絶対禁止は`docs/decisions/rag-memory-privacy-policy-2026-07.md`を正本とする。

## 記憶の分類

### 短期記憶

現在の会話や直近の作業文脈。

用途:

- 会話の流れを保つ
- 直前の指示を理解する
- 作業中の状態を保持する

### 中期記憶

一定期間参照する作業ログや未完了タスク。

用途:

- 今週の農作業
- 最近作った料理
- 進行中の開発タスク
- 配信準備

### 長期記憶

継続的に保持する重要な情報。

用途:

- ユーザーの好み
- 農地・作物の基本情報
- 定番レシピ
- 光織の人格設定
- 継続プロジェクト

## 保存対象

保存してよい情報は、ユーザーが継続参照を望む内容や、後続の支援で明確に価値がある記録に限る。

慎重に扱う情報の認識語彙は設定ファイルで管理する。設定は絶対禁止を削除したり、
保存許可へ反転したりできない。

## 保存時の方針

重要な長期記憶に残す場合は、原則として確認する。

農業日誌やレシピなど、ユーザーが明確に記録を依頼した場合は保存対象とする。

## 記憶の参照方針

応答時には、必要に応じて過去の記録を参照する。

ただし、記憶に存在しない内容を知っているように言わない。

記憶に存在しない内容を推測で補わず、記録に基づく範囲だけを応答に反映する。

## 削除・訂正

ユーザーが記憶の削除や訂正を依頼した場合は、優先して対応する。

削除・訂正に関する具体的な操作は、実装側の機能として扱う。

## 農業日誌の記憶

日付、作物、作業内容、天候、次回作業予定など、後から参照して役立つ文脈を重視する。

## レシピ管理の記憶

料理名、材料、手順、評価、次回試したい変更など、再利用できる文脈を重視する。

## 配信ログの記憶

配信内容、話題、反応、改善点、次回やりたいことなど、継続改善に役立つ文脈を重視する。

## 光織人格の記憶

人格設定は会話ログとは別の基本設定として扱い、通常の長期記憶とは混在させない。

## RAG構築時の考慮事項

光織の「関係の育ち方」を実現するために、以下を考慮する。

- 記録には日付を必ず付与し、時系列での照合を可能にする
- 同時期・同作物・同レシピの過去ログを引き出せる検索設計にする
- 日付差分が必要な応答では、記録の日付を根拠にする
- 参照できる過去ログが増えることで応答の深みが自然に出るようにする
